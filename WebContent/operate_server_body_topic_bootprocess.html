BIOS :-

  • Basic I/O System
  • Independent
  • Primary function to find and execute the boot loader.
  • Perform POST (Power on self test) - Check basic check of the hardware.
  • After POST success only BIOS will check for boot devices.
  • Boot devices ( HDD or USB or DVD or etc)
  • Once Boot loader found, the BIOS will run the boot loader.

Boot Loaders :-

  • Typically GRUB boot loader will use.
  • GRUB ( Grand Unified Bootloader)
  • Boot loader starts the OS
  • Boot loader can start OS with different options like single user,etc..

Initial RAM disk :-

  • Initrd - temporary file system that is loaded from disk and stored in memory.
  • This contains modules required to load the permanent OS file system.
  • Once root file system mounted and OS will continue to run the remaining OS.

/boot directory :-

  • Contains the files required to boot linux.
  • Initrd
  • Kernel
  • Boot loader configuration.
  • #cd /boot; ls -l

Run Levels :-

  • 0 - Sutdown
  • 1,S,s - Single user mode
  • 2 - Multi user mode with graphics( debian/ububtu)
  • 3 - Multi user mode with text (RHEL/Centos)
  • 4 - Undfined
  • 5 - multi user mode with GUI ( RHEL/CentOS)
  • 6 - Reboot

Init :-

  • Generally Run lelvel are controlled by init program
  • Edit /etc/inittab for changing run level.
  • This will start one time at a time
  • This is on old OS.latest replaced with systemd

Systemd :-

  • Parallel booting
  • Systemd will use target instead of run level.
  • #cd /etc/systemd/system ; ls -l *target*
  • #systemctl set-default graphical.target ==> change default run level target
